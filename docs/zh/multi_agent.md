# 多智能体编排

编排指的是应用中智能体的流转：哪些智能体运行、以何种顺序运行、以及它们如何决定接下来发生什么。编排智能体主要有两种方式：

1. 让 LLM 做决策：利用 LLM 的智能进行规划、推理，并据此决定要采取的步骤。
2. 通过代码编排：使用代码来决定智能体的流转。

你可以混合使用这些模式。每种模式都有各自的权衡，详见下文。

## 通过 LLM 编排

智能体是配备了指令、工具和任务转移的 LLM。这意味着针对一个开放式任务，LLM 可以自主规划其解决方案，使用工具执行操作并获取数据，并通过任务转移把子任务委派给子智能体。比如，一个研究类智能体可以配备以下工具：

- 网络检索以在线查找信息
- 文件检索与内容提取以搜索专有数据和连接
- 计算机操作以在计算机上执行动作
- 代码执行以进行数据分析
- 任务转移到擅长规划、撰写报告等的专业智能体

当任务是开放式且你希望依赖 LLM 的智能时，这种模式非常适合。关键做法包括：

1. 打磨高质量的提示词。明确可用的工具、如何使用它们，以及必须遵循的参数范围。
2. 监控你的应用并持续迭代。观察问题出现的地方，并迭代优化提示词。
3. 允许智能体自省与改进。例如，在循环中运行并让其自我批评；或提供错误信息并让其改进。
4. 使用在单一任务上表现突出的专业智能体，而不是期望通用智能体在所有方面都很擅长。
5. 投入于[评测](https://platform.openai.com/docs/guides/evals)。这可以帮助训练你的智能体，在任务上不断改进。

## 通过代码编排

虽然通过 LLM 编排很强大，但通过代码编排能在速度、成本和性能方面让任务更加确定且可预测。常见模式包括：

- 使用[structured outputs](https://platform.openai.com/docs/guides/structured-outputs)生成你可用代码检查的格式良好的数据。例如，你可以让智能体把任务分类到若干目录中，然后基于该目录选择下一个智能体。
- 通过将一个智能体的输出转换为下一个智能体的输入来串联多个智能体。你可以把撰写博客文章这样的任务分解为一系列步骤——做研究、写大纲、写正文、批判性审阅、然后改进。
- 让执行任务的智能体与评估并提供反馈的智能体一起在 `while` 循环中运行，直到评估者认为输出通过了某些标准。
- 并行运行多个智能体，例如通过 Python 基本组件如 `asyncio.gather`。当你有彼此无依赖的多个任务时，这有助于提升速度。

我们在[`examples/agent_patterns`](https://github.com/openai/openai-agents-python/tree/main/examples/agent_patterns)中提供了若干代码示例。
